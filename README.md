Mobile Software Engineer at **Softoio LLC**, building and maintaining production iOS and Android applications with React Native. My work covers the full lifecycle: architecture, implementation, deployment, and keeping production systems stable and reliable.

I work across social platforms, e-commerce, and internal tooling. Most of what I ship is proprietary, but the problems are real: performance under load, Arabic/English bidirectional rendering, release pipelines, third-party integrations, and debugging production issues that don't reproduce locally.

---

## What I work on

```
Architecture         →  Feature-based structure, custom hooks, clean separation of concerns
Performance          →  Optimizing image-heavy feeds and high-traffic screens, render profiling, handling large datasets
Production Debugging →  Root-cause analysis, structured logging, network tracing
Release Management   →  App Store & Play Store, OTA updates, store compliance, CI/CD
Integrations         →  REST APIs, Pusher, Firebase, payment gateways, OneSignal, deep linking / Universal links
Internationalisation →  Arabic-first multilingual apps, RTL/LTR layouts, BiDi text handling
```

---

## Production challenges I've solved

- Feed performance on large API payloads: profiled and cut render time on image-heavy, high-traffic screens
- RTL/BiDi rendering edge cases: paragraph-level direction detection preserving mentions, hashtags, and links
- App Store review and compliance issues: rejections, policy changes, and resubmission cycles
- Production-only network failures: iOS/Android connectivity issues behind Cloudflare that didn't surface in staging
- OTA update strategies: hotfix delivery without full store releases
- Inheriting broken codebases: stabilising and improving systems mid-flight without disrupting what's live

---

## A few things worth knowing

- I'm often the only mobile engineer on a project, which means I own decisions end-to-end: not just the implementation
- A lot of my work involves inheriting codebases mid-flight, understanding them quickly, and improving without breaking what's live
- I care about performance because users notice it, not because it's interesting to profile
- Bilingual in Arabic and English: this shapes how I think about internationalisation, not just how I communicate
